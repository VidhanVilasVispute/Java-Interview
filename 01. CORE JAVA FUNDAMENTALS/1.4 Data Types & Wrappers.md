
49. What are wrapper classes? Why do we need them?

Wrapper classes convert primitive data types into objects. They are needed because many Java APIs, collections, and generics work only with objects. They also provide useful utility methods and can hold null values.

51. Why do we use wrapper classes in collections?

Collections store only objects because generics support reference types, not primitives. Therefore, wrapper classes such as Integer and Double are used to store primitive values in collections.
    
53. Can you explain unboxing and autoboxing?

Autoboxing is the automatic conversion of a primitive into its corresponding wrapper object, while unboxing is the automatic conversion of a wrapper object back into its primitive value. These conversions are performed automatically by the Java compiler.
  
55. Can autoboxing lead to unexpected behavior or NullPointerException?

Yes. Autoboxing itself is safe, but unboxing a null wrapper object causes a NullPointerException because Java attempts to call methods like intValue() on a null reference. Always check for null before unboxing.
