## 21. Explain public static void main(String[] args).

public static void main(String[] args) is the entry point of a Java application. The JVM starts execution by calling this method. public allows the JVM to access it, static lets it be called without creating an object, void means it returns no value, main is the predefined method name recognized by the JVM, and String[] args is used to receive command-line arguments.

## 22. Can we write static public void main() instead of public static void main()?

Yes. static public void main(String[] args) is completely valid. Java does not require public to come before static. However, by convention, developers use public static void main(String[] args) because it is the standard format used throughout the Java ecosystem.

## 23. What will happen if we don't declare the main method as static?

If the main() method is not declared as static, the program may compile, but it will not run because the JVM invokes main() without creating an object of the class. Since a non-static method requires an object, the JVM cannot call it. Therefore, the entry-point method must be declared as public static void main(String[] args).

## 24. Can we override the main method? Can we overload the main method? Can JVM execute our overloaded main method?

The main() method cannot be overridden because it is static, and static methods are hidden rather than overridden. However, it can be overloaded by defining additional main() methods with different parameter lists. The JVM always starts execution with the public static void main(String[] args) method. Any overloaded main() methods are not invoked automatically and must be called explicitly from code.

## 26. If no command-line arguments are passed, what is the value in String[] args?

If no command-line arguments are passed, String[] args is an empty array, not null. Therefore, args.length is 0, and accessing args[0] throws an ArrayIndexOutOfBoundsException.

## 28. Can a .java file be empty? Is it valid?

Yes. A .java file can be empty and is considered valid. It compiles without error, but since it contains no class, interface, enum, or record, the compiler does not generate a .class file.

## 29. Can we execute a program without the main() method?

For a normal Java application, No. The JVM requires public static void main(String[] args) as the entry point. Without it, the application cannot start. However, classes without a main() method can still execute when they are invoked by frameworks such as JUnit, Servlets, Spring Boot, or JavaFX.
