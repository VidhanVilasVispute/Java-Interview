## 21. Explain `public static void main(String[] args)`.

`public static void main(String[] args)` is the entry point of a Java application. The JVM starts program execution by invoking this method.

- **public** – Allows the JVM to access the method from anywhere.
- **static** – Enables the JVM to call the method without creating an object of the class.
- **void** – Indicates that the method does not return any value.
- **main** – The predefined method name recognized by the JVM as the application's entry point.
- **String[] args** – Stores the command-line arguments passed to the program.

---

## 22. Can we write `static public void main()` instead of `public static void main()`?

Yes. `static public void main(String[] args)` is completely valid because Java does not enforce the order of access and non-access modifiers.

However, the standard convention is to write:

```java
public static void main(String[] args)
```

This is the format used throughout the Java ecosystem and is recommended for better readability and consistency.

---

## 23. What will happen if we don't declare the `main()` method as `static`?

If the `main()` method is not declared as `static`, the program may compile successfully, but it will not run.

The JVM invokes the `main()` method without creating an object of the class. Since a non-static method requires an object, the JVM cannot invoke it and reports that the main method is not found in the required format.

Therefore, the entry-point method must be declared as:

```java
public static void main(String[] args)
```

---

## 24. Can we override the `main()` method? Can we overload the `main()` method? Can the JVM execute an overloaded `main()` method?

The `main()` method **cannot be overridden** because it is a **static** method. Static methods are **hidden**, not overridden.

However, the `main()` method **can be overloaded** by defining multiple `main()` methods with different parameter lists.

The JVM always starts execution by invoking:

```java
public static void main(String[] args)
```

Any overloaded `main()` methods are **not executed automatically**. They must be called explicitly from the standard `main()` method or another method.

---

## 26. If no command-line arguments are passed, what is the value of `String[] args`?

If no command-line arguments are provided, `String[] args` is **an empty array**, not `null`.

Therefore:

- `args.length` is `0`.
- Accessing `args[0]` throws an `ArrayIndexOutOfBoundsException`.

---

## 28. Can a `.java` file be empty? Is it valid?

Yes. A `.java` file can be completely empty, and it is considered valid.

The Java compiler compiles it without any errors. However, because the file contains no class, interface, enum, or record, no `.class` file is generated.

---

## 29. Can we execute a program without the `main()` method?

For a **normal Java application**, **No**.

The JVM requires the following method as the application's entry point:

```java
public static void main(String[] args)
```

Without this method, the JVM cannot start the application.

However, classes without a `main()` method can still be executed when they are managed by frameworks or containers such as:

- JUnit
- Servlets
- Spring Boot
- JavaFX

In these cases, the framework provides the entry point and invokes the appropriate methods instead of relying on `main()`.
